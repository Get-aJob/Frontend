# 🎨 스타일 및 공통 컴포넌트 리팩토링 가이드

## 1. index.css (공통 디자인 시스템)

전체 앱의 일관성을 위해 핵심 디자인 토큰을 정의했습니다.

- **폰트**: 영문/숫자 `Inter` + 한글 `Noto Sans KR` 조합으로 세련된 비즈니스 룩 구현
- **컬러**:
  - `--color-bg-view`: `white` (콘텐츠 배경 흰색)
  - `--color-btn-point`: `#8b5cf6` (메인 보라색)
  - `--color-status-success/error`: 상태 표시용 공통 컬러
- **텍스트**: `text-title`(20px), `text-subtitle`(16px), `text-body`(12px)로 규격화
- **유틸리티**:
  - `.btn-point-interaction`: 호버 시 1.05배 확대 (Button 컴포넌트 내장)
  - `.animate-skeleton`: 로딩 애니메이션
  - `.text-ellipsis-1, .text-ellipsis-2`: 텍스트 말줄임 처리 (표준 `line-clamp` 적용)

## 2. Components 수정 내역

### 2-1. common 폴더 리팩토링

#### 📂 Sidebar

- **Lucide 아이콘**: 기존 이모지를 모두 `lucide-react` 라이브러리로 교체
- **SidebarLogo**: 로고 영역에 브랜드 컬러 및 `text-title` 적용
- **SidebarNavItem**: 활성화 시 보라색 포인트 적용 및 텍스트 크기 `text-sm`(14px) 통일
- **SidebarAuth**: 로그아웃 시 `ConfirmModal`을 통해 확인 절차를 거치며, 처리 중 스피너 애니메이션을 노출하도록 개선

#### 📂 Topbar

- **컴포넌트화**: 하드코딩된 버튼과 뱃지를 공통 UI 컴포넌트로 교체
- **z-index 최적화**: 모달 배경(Backdrop)이 Topbar를 정상적으로 덮을 수 있도록 `z-50`으로 조정
- **LoginToast**: 디자인 개선 및 공통 버튼(`Button`) 적용

#### 📂 UI (신규 공통 컴포넌트)

팀원 간 UI 파편화를 막기 위해 `src/components/common/UI`에 생성되었습니다.

- **Button**: `variant`(primary, outline, ghost) 및 `size` 지원. `isLoading` 프롭을 통해 스피너 노출 기능 내장
- **Badge**: 상태 및 태그 표시용(point, success, error, warning)
- **Input**: 모든 폼에서 사용하는 공통 입력창. `Focus` 시 보라색 테두리와 링 효과 자동 적용
- **Skeleton**: 데이터 로딩 중임을 보여주는 반짝이는 박스 (LCP 최적화)
- **EmptyState**: 데이터가 없을 때 사용하는 안내 가이드
- **ConfirmModal**: 확인/취소가 필요한 팝업용 범용 모달. `z-9999`로 설정되어 모든 요소를 덮음

---

### 2-2. Auth (로그인 & 회원가입)

- **로딩 UI 적용**: 로그인 및 회원가입 시 버튼 내 스피너 노출 및 중복 클릭 방지 처리
- **에러 메시지 시각화**: 서버 에러 발생 시 `alert` 대신 폼 상단에 빨간색 에러 박스로 메시지 노출
- **구글 로그인 연동**: 구글 로그인 처리 중에도 전체 폼에 로딩 상태를 공유하여 입력창 및 버튼 비활성화
- **AuthLayout**: 공통 디자인 토큰을 적용하여 배경 장식 컬러 및 텍스트 규격 최적화

### 2-3. Calendar (일정 및 취업 관리)

- **디자인 시스템 통합**: 하드코딩된 보라색 계열 스타일을 전역 변수인 `btn-point`로 교체하여 브랜드 컬러 일관성 유지
- **레이아웃 및 시각적 위계 개선**: 헤더를 재구성하여 연/월 정보와 조작 버튼 영역 분리. '월/주/일' 드롭다운 크기를 배지와 통일하여 수평 균형 확보
- **일정 필터링 기능 도입**: '마감일', '지원 완료' 등 유형별 필터 바를 추가하고, 필터 활성화 상태를 `Badge` 컴포넌트로 시각화
- **상세 보기 및 모달 레이어링**: `CalendarDetailSlide`의 `z-index`를 최적화하여 모달 간 계층 정리 및 텍스트 말줄임 처리 적용

### 2-4. Status (지원 현황 관리)

- **칸반 보드(Kanban Board) 도입**: 기존 리스트 구조를 드래그 앤 드롭이 가능한 칸반 형태로 리팩토링하여 직관적인 상태 관리 구현
- **상태 요약 카드(StatusSummary)**: 상단에 핵심 지표(전체 지원 수, 단계별 현황)를 배치하여 요약 정보 제공
- **D-Day 배지 시스템**: 마감 임박 정도에 따라 `error` 등 컬러 변형이 적용되는 배지 시스템 구축
- **인터랙티브 뷰 전환**: 칸반과 리스트 뷰 간의 토글 기능을 구현하고, 전환 시 부드러운 `fadeUp` 애니메이션 적용
- **StatusDetailSlide**: 맥락을 유지하는 우측 슬라이드 방식 상세 보기 적용 및 `useStatusStore` 연동 메모 기능 강화

### 2-5. Posting (채용 공고 관리)

- **데이터 소스 분리**: '자동 수집', '수동 등록', '직접 입력' 소스를 엄격히 분리하여 탭 메뉴 구성
- **출처별 동적 필터링**: 서버 응답 데이터 기반으로 채용 사이트별(원티드, 점핏 등) 필터 버튼 자동 생성
- **PostingNotice & EmptyState**: 6시간 주기 자동 업데이트 안내 및 검색 결과 부재 시 맞춤형 가이드 제공
- **낙관적 업데이트 댓글 시스템**: `useJobComment` 훅과 Zustand를 결합하여 댓글 작성 시 대기 시간 없는 즉각적인 피드백(Optimistic Update) 구현
- **데이터 매핑 파이프라인**: `mapApiToFeed` 매퍼 함수를 통해 백엔드의 Snake/Camel Case 혼재 문제를 해결하고 데이터 정규화 성공

### 2-6. Resume (이력서 관리 페이지)

- **React Query 연동**: 서버 사이드 상태 관리를 통해 안정적인 이력서 목록 조회 및 동기화
- **인라인 제목 수정**: `UpdateTitleForm`을 통해 페이지 이동 없는 즉각적인 제목 수정 UX 구현
- **빠른 PDF 추출**: `ResumeDownloadButton` 연동으로 목록에서 즉시 PDF 생성 및 다운로드 기능 제공

### 2-7. ResumeForm (이력서 작성 에디터)

- **FormProvider 활용**: 복잡한 다중 섹션 폼을 `react-hook-form`의 `FormProvider`로 관리하여 리렌더링 최적화 및 가독성 확보
- **플로팅 뷰 포트**: 스크롤 대응 유리 질감 상단 바를 적용하여 언제든 저장/미리보기에 접근 가능하도록 설계
- **고급 입력 컴포넌트**: `autosize` 텍스트 에어리어, 글자 수 카운터, 한국어 패치된 `DatePicker` 연동
- **멀티미디어 처리**: `react-dropzone` 기반 PDF 업로드 및 모달 미리보기 기능 탑재

### 2-8. Scrap (저장된 공고 관리)

- **우선순위 렌더링**: 마감 기한에 따른 동적 배지 컬러링 및 '마감 임박순' 정렬 기능 제공
- **로고 플레이스홀더**: 기업 로고 부재 시 기업명을 기반으로 고유 그라데이션 아이콘을 자동 생성하여 시각적 공백 제거
- **레이아웃 최적화**: `max-w-7xl` 규격을 활용해 많은 스크랩 정보를 쾌적하게 조회 가능하도록 설계

### 2-9. Settings (계정 설정)

- **실시간 프로필 동기화**: 이미지 업로드 시 `useAuthStore`와 연동하여 앱 전체의 유저 프로필 즉시 업데이트
- **비밀번호 재설정 보안**: 기존 인증 토큰과 연동된 안전한 비밀번호 변경 프로세스 및 실시간 유효성 검사 로직 구축
- **가입 경로 시각화**: 구글/일반 회원 등 가입 방식에 따른 배지 표시 및 계정 탈퇴/로그아웃 액션 강화

### 2-10. JobPostForm (공고 등록 및 수정)

- **스마트 크롤링**: `CrawlBar`를 통한 URL 파싱으로 공고 정보 자동 입력 기능 구현 (입력 편의성 극대화)
- **유연한 마감일 로직**: 특정 날짜 선택과 '상시 모집' 토글을 동시 지원하는 `DeadlineInput` 구현
- **Zod 유효성 검사**: 실시간 스키마 검증을 통해 정확한 데이터 등록 유도 및 사용자 피드백 강화

### 2-11. Dashboard (통계 분석)

- **Recharts 시각화**: 지원 현황 데이터를 막대, 선형, 파이 차트로 변환하여 활동 성과를 한눈에 파악 가능
- **지표 도출 로직**: 전체 지원 대비 성공률, 진행 중인 공고(Active) 수치 등을 계산하는 전용 가공 로직 구축
- **컴포넌트 독립화**: `StatCards`, `MonthlyChart`, `AnalysisCharts`로 관심사를 분리하여 유지보수성 향상 및 4xl 라운딩 규격 적용
